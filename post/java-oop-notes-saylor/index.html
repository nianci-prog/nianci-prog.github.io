Saylor Academy Overview

Saylor Academy, founded by Michael J. Saylor in 2008, is a nonprofit online education platform headquartered in Washington, D.C., USA. It offers free, self-paced online courses covering humanities, business, mathematics, and science. Students can earn digital certificates, and some courses provide ACE-accredited college credits. By 2025, Saylor Academy has served over 2.5 million students worldwide.

Learning Reflections

Studying Java object-oriented programming (OOP) helped me understand modularity and maintainability in programming. Through classes, objects, inheritance, encapsulation, abstraction, and polymorphism, I learned to design flexible, reusable program structures and map real-world concepts to code. Implementing examples like SUV, Student, and Calculator taught me how parent and child classes interact, how to override methods, and how to protect data, improving program organization and extensibility.

Java OOP Key Notes

OOP Features

Feature Description Java Support
Abstraction Focus on “what” not “how” abstract class, interface
Encapsulation Bundle data and methods class, object
Information Hiding Hide internal details, expose interface private, public, getter/setter
Inheritance Subclass inherits parent attributes/methods extends, implements
Polymorphism Same method, different implementations method overriding, polymorphic calls

Class vs Object

Term Associated With
Abstraction Class
Member Class
Template Class
Design Class
Module Class
Information Hiding Class
State Object
Message Object

Static Keywords

Statement True / False
Static data/method belongs to class True
Static data/method belongs to object False
Static class data shared by all objects True
Static class data copied per object False
Static methods can call only static members True
Constructor can be static False

Abstract Class

Feature Description
Declaration Use abstract class
Can instantiate No
Content Can contain abstract (no implementation) and concrete methods (with implementation)
Purpose Enforce subclass implementation; suitable for partial shared and partial customized logic

Class/Object/Subclass Data Changes

Change Type Effect
Class data value changed Affects all objects
Object data value changed Only affects that object
Subclass data value changed Only affects that subclass

Examples

1. Vehicle → SUV

Class Attributes / Methods
Vehicle Attributes: weight, color, numberOfWheels; Methods: setWeight, setColor, setNumberOfWheels, toString()
SUV (extends Vehicle) New attributes: make, model, is4wd; Methods: setMake, setModel, setIs4wd, toString() (calls super.toString())
Driver Test Create suv1, set attributes, output full info

2. Array1dProc

Method Function
arraysum() Return sum of array
arraymax() Return max value
printelements() Print all elements
main() Read user input, create Array1dProc, output sum and max

3. BasicCalculator → AdvCalculator

Class Attributes / Methods
BasicCalculator Attributes: z, txt (private → use getter/setter); Methods: plus, minus, mult, divide
AdvCalculator (extends BasicCalculator) Adds methods: power(x, y), nthroot(x, y)
TestCalc Read input → test addition, subtraction, exponentiation, root

4. Person → Student

Class Attributes / Methods
Person Attributes: firstName, lastName, idNum, birthdate; Methods: setter, toString()
Student (extends Person) Attributes: gpa, creditHours, level; Methods: setGPA, setCrHrs, setLevel, toString() (calls super.toString())
Driver Test Create student1, set attributes, output full info

5. Property → House / Building

Class Attributes / Methods
Property Common property attributes & methods
House / Building Subclasses adding specific attributes & methods
