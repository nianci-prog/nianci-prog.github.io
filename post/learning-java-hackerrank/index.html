  • Comments:

    • Multi-line: /* ... */
  • Semicolons: End each statement with ;

  • Variable declaration (int)

3. Abstract Classes and Inheritance

abstract class Calculator {
    abstract int add(int a, int b);
}

class Adder extends Calculator {
    @Override
    int add(int a, int b) {
        return a + b;
    }
}

public class Solution {
    public static void main(String[] args) {
        Calculator adderObject = new Adder();
        System.out.println("The sum is: " + adderObject.add(5, 7));
    }
}
  • abstract class and method
  • extends for subclassing
  • @Override annotation for method implementation

4. Scanner Input

import java.util.Scanner;

public class Solution {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);
        int a = scan.nextInt();
        int b = scan.nextInt();
        System.out.println("Sum: " + (a + b));
        scan.close();
    }
}
  • Use Scanner to read user input
  • nextInt() to read integers
  • Always close() the scanner

5. Bitwise Operations and Comments

int mask = 0x000F;       // 0x000F = 0000 0000 0000 1111
int value = 0x2222;      // 0x2222 = 0010 0010 0010 0010
int result = value & mask; // Bitwise AND
System.out.println(result); // Output: 2
  • Single-line and multi-line comments
  • Comments may include symbols and numbers
  • Note: Multi-line comments cannot be nested

6. Map Interface and Implementations

Map Type Ordering Allows Null Thread-Safe Notes
HashMap None One null key, multiple null values No Fast and commonly used
LinkedHashMap Insertion order Yes No Preserves order
TreeMap Sorted by key No No Slower, automatically sorted
Hashtable None No Yes Legacy implementation
ConcurrentHashMap None No Yes High-performance thread-safe

Example:

Map<String, Integer> scores = new HashMap<>();
scores.put("Alice", 90);
scores.put("Bob", 85);
System.out.println(scores.get("Bob")); // Output: 85
